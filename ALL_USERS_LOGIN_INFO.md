## Important Notes

1. **Passwords:** All passwords are hashed using bcrypt and cannot be retrieved from the database.

2. **Login Method:** Users can login using either their **username** OR **email address**.

3. **Password Reset:** If users forget their password, they must use the password reset feature in the application.

4. **Known Password:** Only the SuperAdmin user (`superadmin`) has a known default password: `Admin@123456` - this should be changed immediately for security.

5. **Status:** 
   - ✅ Active = User can login
   - ❌ Inactive = User account is disabled
   - ✅ Verified = Email verified
   - ❌ Not Verified = Email not verified

6. **Role Assignment:** Some users have both legacy role (ADMIN, DOCTOR, etc.) and new role system (SuperAdmin, Medico, etc.). The new role system takes precedence.
